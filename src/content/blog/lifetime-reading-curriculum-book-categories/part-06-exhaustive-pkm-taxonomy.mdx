# Part 6: The Exhaustive PKM Book Taxonomy

> **A PKM without a comprehensive taxonomy is just a digital junkyard.**
> To build a library that scales to thousands of volumes, you need a highly
> structured, three-level hierarchy. This article maps out an exhaustive,
> zero-overlap taxonomy designed to organize all books and notes.

---

## The Core Design Principles

Before diving into the categories, let's examine the three structural rules
that govern this taxonomy:

1. **Meta-Learning First**: To master any subject, you must first master the
   tools of acquisition. Category 01 contains learning theory, reading techniques,
   note-taking systems, and memory retention. Read these books first.
2. **Zero-Overlap Merging**: Philosophy and Religion are two sides of the same
   existential coin. Merging them into a single category avoids the common
   confusion of where to place books like the *Bhagavad Gita* or *Meditations*.
3. **Three Levels of Nesting**: The folders nest exactly three levels deep. The
   fourth level holds the book folder itself, structured as
   `[Book Title] by [Author Name]`.

---

## Folder Architecture

The directory structure on your filesystem or Git repository looks like this:

```
books/
└── [Category_01]/
    └── [Subcategory_01]/
        └── [Sub-subcategory_01]/
            └── [Book Title] by [Author Name]/
                ├── index.mdx            ← Synopsis, ratings, and tags
                ├── analysis.mdx         ← Deep critical dissection
                ├── narration.mdx        ← Chapter-by-chapter summaries
                └── metadata.json        ← Structured metadata (JSON)
```

## Disambiguation Guidelines

When a book fits multiple folders, resolve the placement with these four rules:

1. **Primacy of Action**: Place the book under the folder where you will apply
   the knowledge. A book on negotiating tech deals goes to **Communication &
   Writing → Interpersonal Communication**, not Law.
2. **First-Read Preference**: If a book provides significant general learning
   heuristics (e.g., *How to Read a Book*), place it in **Category 01**, even
   if it discusses literature or history.
3. **Primary Focus Rule**: Where does the author spend more than 50% of the
   pages? That domain wins.
4. **Metadata Backlink**: Always place the book in one physical directory, but
   reference secondary categories via the `tags` array in `metadata.json`.
